6. Nerverun 类型修正。结构化档位同时包含永久余额和订阅套餐，因此站点类型推断为混合型，前端类型列不再显示“待补证据”。
7. Nerverun 未入榜原因修正。用户继续指出费用证据已抓到，不应显示“缺正式费用行”；最终口径为费用证据已具备但请求样本为 0，所以显示“缺请求样本”。

## 用户纠偏点

- “最低倍率”是未排名表的展示增强，不影响正式排名和评分。
- Relay 的充值证据应以订阅 plans 的一次性档位为准，余额 status 不能覆盖它。
- 分组没有明确写 Claude 或国产/公益时，应默认可作为 Codex-like 候选，不能只匹配 `codex/openai/gpt/default`。
- Nerverun 已抓到有效充值和订阅证据，类型应自动推断为混合型。
- Nerverun 缺的是请求样本，不是费用证据；未入榜原因必须把这两类缺口分开。

## 经验

- 详情页证据完整不等于可入榜；正式排名仍必须同时有请求样本和可采用费用证据。
- 未入榜原因应优先帮助定位真实缺口：费用证据齐全但样本为 0 时写“缺请求样本”，不要用“缺正式费用行”遮住问题。
- 公开 status、订阅 plans、登录态支付 API、官方店铺的权威边界要分清；同一站点有多个入口时，应以更贴近真实充值/购买动作的结构化来源为准。
- 宽松 Codex-like 规则更贴近实际站点命名，但必须保留 Claude、Kiro、国产和公益模型排除，避免被非 Codex 通道拉低成本。
- 单站补抓应只写该站 probe，并用结构化 diff 确认其他站点不变；页面核验用于确认脚本结果，不替代数据生成流程。
